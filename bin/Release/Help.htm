Installation.

Execute the setup program.

Implementing a backup system.

Before you can create your first backup, you’ll have to create one (or more) scripts, explaining to the software, what you want to include or exclude from the backup, and where to place the copy. To do so, run the program:


and create a "new backup script" (That you need to look for a button on the screen is indicated with the blue colour). You’ll be presented a list of all root folders on each drive in the pc.


Select the drives, that you wish to have in the backup (click for the first and ctrl-click or shift-click for the others). Click on OK. If you don’t want to backup entire drives, you may now add single folders by clicking on "Browse". Next you'll have to specify the destination of the backup. To do so click “Destination...” and browse to the folder. Clicking on “Options”, you have the possibility to change default behaviour. See below for details.

After having given all this information, the program is ready to proceed in 2 different ways:

  1. For beginners

  2. For experienced users.

For beginners or inexperienced users:

Click on OK and the program will list all folders with files that are to be backed up:


You may expand lines to see more detail. Now start excluding all unwanted folders and/or files by right clicking the mouse and choosing from the list what to to


Choose “Exclude name” on a folder of a file name to exclude this folder or file from the backup. Choose the “everywhere” version to exclude this name where ever it exists on your pc. If the name you chose contains a dot (.) you'll be offered the “Exclude type” option: all names at the same folder level as the name selected (those above and underneath indented as the selected one) that have the same type (the text in the name after the last dot) will be excluded. Again choose the “everywhere” version to exclude this type where ever it exists on your pc. This last options is practically useful to exclude all non relevant files such as .log or .tmp files. Find one of those files and with one mouse click each of them will be excluded from the backup. Note that when you have to choose from a menu, I use the red colour.

When the list contains only folders and files that you wish to back up, click right and choose the “Start backup” option to actually backup. The “Go to expert mode” is explained for experts, below. Before backing up, first you'll have to save the script that you have just composed.


Choose a name and a folder that suits you well.

The backup process will start, showing the progress at the top, expanding little by little the tree structure. If a file exceeds the size, specified in the Options screen, the usual copy dialogue will appear, indicating progress and time remaining.


If any file could not be copied, it appears in a list, with the reason:


Select any files, that you don't want to backup in the future and select ”Delete n items” with a right click to add these names to the list of excludes files. At the end of the backup, close any window, to finish the program.

The next time that you want to run a backup, on the initial screen choose “Open Backup script” and select the name you used to save the script. You'll see the script that you created before. Don't look at it and click “Analyse” if you would like to exclude any accidentally included folder and follow the procedure as described above, or click “Backup” to immediately run the backup.




For experienced users:

Proceed as above for the first 2 screens and then click on “OK and expert mode” To determine what to back up, the program knows 2 different strategies: an “include” and an “exclude” strategy. For the selected drives or folders, and for any folder that you name further on, you need to ask yourself: "Shall I include only specified folders (include strategy), or shall I include everything, except a list of excluded folders (exclude strategy)?". The main difference stands in the fact, that in the latter case, any new folder will automatically be included in the backup. If you do not specify any rule (include, select or exclude), the folder and all it’s subdirectories will be included. If you want to include a sub-folder (using include or exclude), you may select intermediate levels in the path instead of including them, if you are with the exclude strategy. The effect is that the folder counts neither for include or for exclude and thus will be ignored when determining if the include or if the exclude strategy is followed. For each selected folder you may specify additional include, exclude or select rules. When backing up, the program analyses the rules for each folder separately. If for a folder there exists an include (specific or in de general specifications), only the named folder for inclusion will be processed. If no include exists for the folder all sub folders will be included. From the list of included folders the program next removes all names that conform to any exclude rules. The remaining list will be backed up.

So if for any of the roots selected you don’t want to backup all sub folders, specify details for that folder; to include all sub folders, leave this line as it is. To add a specification, right click on the name and choose “Add”. For each option you will get more or less this screen:


If you decided for the INCLUDE strategy, select "Include folder", then

  • "browse" to find it’s name;

  • or specify a name* or name??? type of string, that confirms to the syntax of the MS-DOS DIR command, to include a group of folders;

  • Check the boxes to include hidden folders (/d in script) or hidden files (/f in script) in this and its sub folders, if you want these included in the backup. The specification inherits the parent boxes.

  • Check the Create incremental backup's box (/i in script) if you want the files in this folder to be copied separately (per change date). Not checking the box will cause overwriting of previous versions of the file. Checking it causes creation of a folder with de file name and in the folder all the different versions of the file

  • Check the Valid for all places box, if you want this inclusion to be considered for every folder that's being processed. It will put the specification in the general part (under root name *), instead of in the currently selected folder.

Click "OK" to confirm..

If you decided for the EXCLUDE strategy, select "Exclude folder", and proceed as above for include.

If you need to proceed along a path, not included nor excluded, select “Select folder”, and proceed as above for include. If you want to exclude ALL folders, and just want to pass here to backup a folder deeper in the structure, you can specify to INCLUDE a folder named “dummy” (all lowercase) in combination with one or more “Select folder” to proceed deeper in the structure. Select dummy inhibits the automatic choice of all folders if no Exclude and no Include is present.

Once you specified an inclusion, selection or exclusion for a folders to be backed up, you should treat it just as you treated the roots. For each folder not entirely to be included, specify what you want, by specifying includes, excludes of selects. A selected folder needs to be specified further, otherwise it makes no sense specifying it.

If you read the script, and see any errors, correct then with the “change” option in the menu, or delete the entry (and everything belonging to it) with “delete”.

Include or exclude files.

As for files to include or exclude, you get this screen:


If you want to includes a file, select “include file” and then

  • "browse" to find it’s name;

  • or specify a name* or name??? type of string, that confirms to the syntax of the MS-DOS DIR command, to include a group of files;

  • Check the Valid for all places box, if you want this inclusion to be considered for every folder that's being processed. It will put the specification in the general part (under root name *), instead of in the currently selected folder

Save the backup script that you created with the "save script", if you want to reuse it.

Run a backup.

To create a new script follow the procedure explained above. To run a script again “open a script“ that you created and saved previously. On the next screen click on “Analyse“ if you want to see which files will be included in the backup or to adjust the previously created script. If you’d see any files or folders, that should be excluded from the backup, just right click on the name and exclude the file or the type. Right click on the analysis screen to start the backup. Click “Backup” immediately after opening a script if you’re not interested in the analysis results. If any file could not be copied, it appears in a list, with the reason. Select any files, that you don't want to backup in the future and select ”Delete n items” with a right click to add these names to the list of excludes files.

Remove deleted files in the backup.

When you want to remove (some) of the files once backed up, but not existing any more as an original, “open a script” that you used to backup. Click on “Confront” to create a list of all files / folders that don’t exist as original. When the list is completed, remove a file or a folder with a right click on the name. Don't forget to clean up the recycle bin to eliminate them completely.

Options.

You may specify options for running the software by clicking on the “options” button:

  • Write a detailed report of the backup process in a .log file, which you should name on the options screen. Use “browse” to navigate on your pc.

  • Verify the correctness of the backup by checking the contents of the copy against the original.

  • Shut down the pc, after the backup ended successfully, and you don’t want to wait for the backup to end. You can specify to shut down only this time.

  • Show the standard Explorer type progress bar when files exceed a given dimension.

When you are defining a new script, there is another option

  • Exclude system folders from the backup:

  • The Windows directory

  • The programs directory

    When creating a new script it is your own choice if you backup the temporary (internet) files.



Restore.

You may restore a file by browsing the directory where the backup is stored, en following the structure, you'll find the latest backup. Copy it using Window's own Explorer.

Implementare un sistema di salvataggio (Backup). 

Per il salvataggio, devi creare una (o più) procedura, spiegando al software cosa vuoi includere o escludere nel salvataggio e dove mettere la copia. Per fare una procedura, lanci il programma


e "Crea procedura di salvataggio" (Che devi cercare un pulsante nella finestra viene indicato con il colore blu). Ti verrà presentato la lista delle cartelle principali del unità del pc.


Selezionare una o più cartelle da salvare (clicca per la prima, ctrl-clic o Shift-clic per altre). Clicca su “OK”. Se non vuoi salvare intere unità, clicca su "Trova" per aggiungere singole cartelle. Dopo devi nominare la cartella dove salvare i dati. Per farlo, clicca su “Destinazione...” e trova la cartella. Se clicchi su “Opzioni”, hai la possibilità di cambiare il comportamento standard del programma. Vedi dopo per dettagli.

Dopo aver fornito le informazioni, il programma può procedere in 2 modi diversi:

  1. Per principianti

  2. Per utenti con esperienza

Per principianti:

Clicca su “OK” e il programma ti creerà una lista di tutte le cartelle e tutti gli archivi da salvare.


Puoi aprire singoli pezzi dell'albero per vedere più dettagli. Adesso cominci a eliminare tutte cartelle e tutti archivi da non salvare, cliccando col tasto destro e facendo la scelta dalle possibilità offerte:


Scegli “escludi nome” per una cartella o un archivio per escludere quel none dal salvataggio. Scegli la versione “dappertutto” per non salvare un simile nome anche se in altri posti nel pc. Se il nome che hai scelto, contiene un punto (.) c'è anche la possibilità “escludi tipo” da escludere tutti nome con lo stesso tipo (il testo dopo l'ultimo punto nel nome) nella cartelle corrente (quelli sopra e sotto con la stessa indentazione del nome selezionato) La versione “dappertutto” offre la possibilità di escludere quel tipo anche in tutti gli altri posti del pc. Questa opzione è più utile con archivi che non con cartelle, credo, per non copiare archivi di tipo .log o .tmp. Trova un archivio di quel tipo per escluderli dappertutto dal salvataggio. Nota, che uso il colore rosso per indicare la scelta di una voce in un menù.

Quando la lista contiene solo cartelle e archivi da salvare, clicca tasto destro e scegli “inizio salvataggio” per iniziarlo. La scelta “vai al modo di esperto” è spiegato qui sotto, e vale per esperti solo. Prima di salvare viene richiesto di salvare la procedura che hai appena finita di comporre.


Scegli un nome che a te va bene.

Se non era possibile di copiare un archivio, il nome appaia in una lista con il motivo.


Selezioni gli archivi, che nel futuro non vuoi più includere nel salvataggio, clicca destro e selezioni “Escludere n archivi”e quelli verranno aggiunti alla lista di archivi da escludere. Alla fine del salvataggio, chiudi le finestre, per completare il programma.

Vedrai la procedura creato l'altra volta. Non guardarla, ma clicca “Analizza” se ci sono altre cartelle da escludere dal salvataggio. O clicca subito “Inizio salvataggio” per salvare i dati.

Per ogni unità selezionata, e per ogni cartella selezionata più avanti,devi chiederti: "Salverò solo cartelle specificate (strategia di inclusione) oppure salverò tutte le cartelle contenute con eccezione di una lista di cartelle da escludere (strategia di esclusione)?". La differenza principale sta nel fatto che nel secondo caso cartelle aggiunte dopo, saranno salvato automaticamente. Se per una cartella non specifichi nessun regola di inclusione o di esclusione, tutte le sotto cartelle verranno incluse. Se vuoi includere una cartella solo in parte, puoi selezionare le cartelle del percorso intermedio invece di includerle. L'effetto è, che la cartella selezionata non conta come inclusa, ne come esclusa, e così non ha interferenze con la strategia scelta e verrà saltata per determinare che strategia stai seguendo. Per ogni cartella selezionata puoi specificare regole addizionali per inclusione, esclusione o selezione. Quando sta facendo il salvataggio il programma analizza le regole cartella per cartelle separatamente. Se per una cartella esiste una inclusione (specifica o nelle regole generali) solo la cartella nominata verrà salvata. Se non esiste nessuna inclusione per la cartella tutte le sotto cartelle saranno incluse. Implementatie van een backup systeem.

Voordat de eerste backup maakt, moet je een of meer scripts maken, die uitleggen aan de software, wat je wel en niet wilt backuppen en waar de backup gemaakt moet worden. Start het programma:


creëer een "Nieuw backup script" (Dat je moet zoeken naar een knop moet zoeken geef ik aan met de kleur blauw). Je krijgt een lijst van alle root mappen op het systeem.


Selecteer er nul of meer van: die schijven, die je vrijwel geheel wilt backuppen (klik voor de eerste en ctrl-klik of shift-klik voor de volgende). Klik op OK. Als je niet hele disks wilt backuppen, voeg dan individuele mappen toe door op "Bladeren" te klikken. Ook moet je de bestemming opgeven. Klik op “Bestemming...” blader naar de map. Als je op Options klikt, kun je de basis werkwijze veranderen. Details worden verderop nog uitgelegd.

Nadat je deze informatie hebt opgegeven, kun je op 2 manieren verder gaan:

  1. Voor beginners

  2. Voor gebruikers met ervaring.

Voor beginners of onervaren gebruikers:

Klik op OK en het programma toont je alle mappen en bestanden die gebackupped kunnen worden:


Je kunt regels in en uitklappen voor meer of minder details. Begin nu alle mappen en bestanden uit te sluiten, die je niet wilt backuppen door met de rechter muis op de uit te sluiten naam te klikken


Kies “Neem niet mee” voor een map of bestand om deze uit te sluiten van de backup. Kies “nergens” om deze naam nergens mee te nemen, in welke map hij ook voorkomt. Als in de naam een punt (.) voorkomt kun je ook kiezen voor “Neem type niet mee”: alle namen, in de map op het zelfde niveau als waarop de geselecteerde naam staat (erboven en eronder en evenveel ingesprongen) zullen niet meedoen als ze hetzelfde type (de tekst na de laatste punt in de naam) hebben. Ook nu kun je “nergens” kiezen om dit type naam nooit mee te nemen. Deze optie is praktisch als je bestanden van het type .log of .tmp niet wilt backuppen. Zoek er een op en sluit hem overal uit.

Zodra de lijst alleen mappen en bestanden bevat, die je wilt veilig stellen, klik rechter muisknop en kies “Start backup” om de backup werkelijk uit te voeren. De keus “Naar expert modus” wordt uitgelegd bij “ervaren gebruikers” hieronder . Vooraf aan de backup wordt je gevraagd om het script ergens te bewaren.


Kies een naam en plaats, die je past.

Het backupproces start en toont bovenaan de voorgang, terwijl de boomstructuur beetje bij beetje wordt geopend. Als een bestand de gekozen waarde in de opties overschrijdt, wordt het standaard voortgangsscherm van Explorer getoond met de tijd die het nog duurt.


Als een bestand niet gekopieerd kon worden, wordt de naam in een lijst geplaatst met de reden:


Selecteer de files die je voortaan niet meer wilt veilig stellen, en kies “Neem n bestanden niet meer mee” onder de rechter muisknop. De namen worden opgenomen als uit te sluiten bestanden. Als de backup klaar is, sluit alle schermen en het programma is klaar.

De volgende keer dat je een backup wilt uitvoeren, kies dan “Open Backup script” op het eerste scherm en kies de naam van het script. Het script wordt getoond maar kijk er niet naar. Klik “Analyseer” als je nog mappen of bestanden extra wilt uitsluiten volgens de boven beschreven procedure, of klik meteen op “Backup” om meteen te starten.




Voor ervaren gebruikers:

Volg eerst de procedure voor beginners voor de eerste 2 schermen, en klik dan op “OK en expert modus”. Om te bepalen, wat je gaat backuppen, kent het programma 2 strategieën: een “meeneem” en een “weglaat” strategie. Voor de geselecteerde schijven en mappen, en voor iedere nog gespecificeerde map om mee te nemen, moet je jezelf afvragen: “Zal ik alleen genoemde mappen meenemen (meeneem strategie), of zal ik alles meenemen behalve de mappen, die ik wil weglaten (weglaat strategie)?". Het basis verschil zit 'm erin, dat in het laatste geval nieuwe mappen automatisch meegaan in de backup. Wanneer je voor een map geen enkele regel specificeert, worden alle mappen in die map, meegenomen in de backup. Als je een deel van een submap wilt uitsluiten, kun je tussenliggende nivo's selecteeren, in plaats van ze mee te nemen, als je de weglaat strategie wilt hanteren. Het gevolg is, dat de map niet voor meenemen meetelt, als je de weglaat strategie wilt volgen. Voor iedere meegenomen of geselecteerde map kun je weer nieuwe regels specificeren met hun eigen strategie. Als voor een map een mee te nemen sub map wordt gespecificeerd ( specifiek of in de algemene regels) dan word(en)t de meegenomen mappen in een lijst geplaatst. Zijn er geen meegenomen mappen, dan worden alle sub mappen in de lijst gezet. Uit de lijst worden de over te slaane mappen gestreept. Van de overgebleven mappen worden de bestanden veilig gesteld.

Dus als je voor een van de root mappen niet alle submappen wilt backuppen, specificeer dan regels, en anders laat je de regel staan zoals ie is. Om een specificatie toe te voegen, rechtsklik op de naam en kies “Toevoegen” (menu opties zijn in rood). Voor elke optie krijg je ongeveer dit scherm:


  • Als je beslist voor de MEENEEM strategie, selecteer dan "Neem map mee", en dan

    • "blader" om de naam te vinden;

    • of specificeer een naam* or naam??? type tekst, die voldoet aan de MS-DOS DIR commando syntaxis, om een groep mappen op te geven;

    • Vink de vakjes om verborgen mappen of bestanden mee te nemen in de backup. De vinkjes worden geërfd van de hogere map.

    • Vink het vakje Maak incrementele backups als je wilt dat de bestanden in deze map apart per datum worden bewaard. Niet vinken betekent, dat de alleen de laatste versie van het bestand bewaard blijft. Een vinkje, dat er een map in de backup gemaakt wordt met daarin per wijzigingsdatum een versie van het bestand.

    • Vink het vakje “Geldt voor alle mappen, niet alleen deze”, als je deze meeneem regel wilt laten gelden voor alle mappen. De specificatie komt in de algemene specificaties (onder root *) te staan in plaats van de actuele map.

      Klik "OK" om te bevestigen.

  • Als je besluit tot de weglaat strategie, selecteer dan "Sla map over", en ga verder als hierboven voor “map meenemen”.

  • Als je een map wilt noemen, omdat verderop in de structuur een specificatie gewenst is, kies dan voor “Selecteer map”, en ga verder als hierboven. If you want to exclude ALL folders, and just want to pass here to backup a folder deeper in the structure, you can specify to INCLUDE a folder named “dummy” (all lowercase) in combination with one or more “Select folder” to proceed deeper in the structure. Select dummy inhibits the automatic choice of all folders if no Exclude and no Include is present.

Zodra je voor mappen de benodigde specificaties voor meenemen, overslaan of selecteren hebt opgegeven, behandel je deze nieuwe mappen als de eerdere mappen: specificeer voor iedere map, die niet geheel mee moet in de backup, wat er dan wel en niet mee moet. Een geselecteerde map moet een specificatie bevatten, anders heeft het geen zin om hem te selecteren.

Als je in het script fouten ziet, corrigeer ze dan met de “Wijzigen” optie in het menu of verwijder de regel met “Verwijder”.

    • "bladeren" om de naam te vinden;

Sla het script op, dat je nu gemaakt hebt, met "sla script op", om het te kunnen hergebruiken.

Een backup uitvoeren.

Om een backup uit te voeren, maak je een Nieuw backup script zoals boven aangegeven of Open backup script dat je eerder hebt aangemaakt en bewaard. Klik op Analyseer als je eerst wilt zien welke bestanden gekopieerd gaan worden en als de lijst klaar is klik je op Backup. Als je nog bestanden of map ziet, die niet mee zouden moeten doen, rechts klik en sla bestand of bestand type over, zoals eerder beschreven. Je klikt meteen op Backup als je zeker bent dat de lijst van bestanden in orde zal zijn. Als een bestand niet gekopieerd kon worden, wordt de naam in een lijst geplaatst met de reden. Selecteer de files die je voortaan niet meer wilt veilig stellen, en kies “Neem n bestanden niet meer mee” onder de rechter muisknop. De namen worden opgenomen als uit te sluiten bestanden.

Verwijderde bestanden uit de backup halen.

Als je bestanden wilt verwijderen die ooit in de backup zijn gezet, terwijl het origineel niet meer bestaat, Open backup script dat je gebruikte om de backup te maken. Klik op Confrontatie om een lijst te maken met alle bestanden en mappen die niet meer in het origineel bestaan. Zodra de lijst klaar is, kun je met een rechter muis klik op de naam een bestand of een map uit de backup weggooien. Gooi ze daarna ook uit de prullenbak weg, als je ze echt definitief wilt verwijderen.

Opties.

Je kunt opties om de software uit te voeren opgeven door te klikken op de Opties knop:

  • Schrijf een gedetailleerd log-bestand, waarvan je de naam hier op moet geven. Klik op bladeren om op je pc de naam en plaats te vinden.

  • Verifieer de volledigheid van de backup, door de inhoud van origineel en van de kopie te vergelijken.

  • Sluit de pc af, nadat de backup klaar is en je niet wilt wachten op het voltooien. Je kunt deze optie alleen voor deze run aangeven.

  • Toon de standaard voortgangs scherm van Explorer als een bestand groter is dan de opgegeven waarde..

Als je een nieuw script aanmaakt, is er een extra optie, om systeem mappen niet mee te nemen:

  • de Windows map

  • de programma's map

    als je een nieuw script maakt is het je eigen keus of je tijdelijke bestanden (Temp) en tijdelijke internet bestanden opneemt in de backup.



Restore.

Je kunt een bestand restoren, door te bladeren in de structuur van de backup map. Daar vind je het bestand, en je kunt het simpel met Window's Explorer copieeren.
